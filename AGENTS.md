# AGENTS.md ‚Äî Parsify.dev

AI-powered SEO analysis tool. Paste a URL ‚Üí **Jina Reader** (`r.jina.ai`) fetches and converts it to LLM-optimized markdown ‚Üí **DeepSeek** `deepseek-v4-flash` generates SEO analysis (SEO.md + robots.txt + llm.txt). Both API keys live on the server. Built with TanStack Router (SPA) + Vite 7 + Hono at `/api/*`, deployed via Dokploy + Docker.

## Architecture

**TanStack Router (SPA mode)** ‚Äî client-side rendered SPA with TanStack Router handling file-based routing. Vite 7 is the build tool.

**API layer**: Hono app (`src/server/hono.ts`) configured with `{ strict: false }` (trailing slashes allowed), `.basePath('/api')`, mounted via a TanStack Router catch-all route (`src/routes/api/$.ts`). All HTTP methods forwarded to `app.fetch()`.

**Server routes:**
- `GET /api/health` ‚Äî returns `{ ok: true }`
- `POST /api/parse` ‚Äî Jina Reader proxy. Validates URL via Zod (SSRF guard rejects private/loopback/link-local hosts). Forwards user prompt as `X-Instruction`. Accepts optional `objective` (‚â§500 chars). Enforces 5 MB markdown response limit server-side.
- `POST /api/agent` ‚Äî DeepSeek SSE proxy. Validates `markdown` (‚â§1 MB) and optional `prompt` (default: SEO analysis prompt). Supports `outputFormat: 'json'` (default, returns structured SEO analysis) and `outputFormat: 'text'` (streaming plain text). Returns 500 if `DEEPSEEK_API_KEY` is missing.
- `GET /api/llm.txt` ‚Äî LLM-optimized description of Parsify SEO Analyzer
- `GET /api/robots.txt` ‚Äî Search engine crawler instructions
- `GET /api/sitemap.xml` ‚Äî XML sitemap for search engines

**Validation**: Zod 4 schemas in `src/schemas/`. The parse schema has SSRF guard ‚Äî add test coverage in `src/__tests__/schemas/parse.test.ts` whenever it changes.

**Logging**: Custom console wrapper with level filtering in `src/lib/logger.ts`. Level controlled by `LOG_LEVEL` env var (default `info`). Includes redaction for `apiKey`, `authorization`, `cookie` patterns. Always use `logger` from `~/lib/logger` ‚Äî never raw `console.log`.

**Rate limiting**: `hono-rate-limiter` on `/api/parse` and `/api/agent` (20 req / 15 min per IP). Single-container deploy assumption; replace store if multi-instance.

## Code style (Biome)

- 2-space indent, 100 char width, single quotes, JSX double quotes, semicolons always, trailing commas (ES5), LF
- `assist.organizeImports: "on"` ‚Äî imports are auto-organized on save
- Enabled rules: `noUnusedVariables`/`noUnusedImports` (error), `noLabelWithoutControl` (off), `noForEach`/`useLiteralKeys` (off), `noNonNullAssertion`/`noArrayIndexKey`/`noAssignInExpressions`/`noExplicitAny`/`noImplicitAnyLet` (off)
- TypeScript: strict, `noUncheckedIndexedAccess`, `noPropertyAccessFromIndexSignature`
- `import type` for type-only imports, explicit return types on exported functions

## TypeScript gotcha

`noPropertyAccessFromIndexSignature` means `process.env.NODE_ENV` fails ts(4111). Use bracket notation: `process.env['NODE_ENV']`. Same for any index-signature property access.

## Deploy (Dokploy + Docker)

Push to `main`. Docker build (`Dockerfile`):
1. `bun install --frozen-lockfile`
2. `bun run build` ‚Üí `dist/`
3. `CMD ["bun", "run", "start"]`

**Required env vars:**
| Var | Required | Purpose |
|---|---|---|
| `PUBLIC_ORIGIN` | yes | Canonical origin ‚Äî used for CORS `origin` and sitemap/llm.txt/robots.txt |
| `DEEPSEEK_API_KEY` | yes | `/api/agent` returns 500 without it |
| `JINA_API_KEY` | no | Without it, `/api/parse` uses anonymous tier (20 RPM) |
| `LOG_LEVEL` | no | pino level (default `info`): `trace`, `debug`, `info`, `warn`, `error` |

Note: `.env.example` uses `PUBLIC_ORIGIN=http://localhost:5173` matching the dev server default port.

## Security

**Server-side key invariant**: `DEEPSEEK_API_KEY` and `JINA_API_KEY` are read from `process.env` only. Never sent from browser, never logged, never persisted (no DB, no cache, no file), and never echoed in responses. Logger redaction is defense-in-depth.

- **SSRF guard**: `parseRequestSchema` rejects 127.x, 10.x, 172.16‚Äì31.x, 192.168.x, 169.254.x, ::1, localhost.
- **Rate limiting**: `/api/parse` and `/api/agent` capped at 20 req / 15 min per IP.
- **Logging**: use `logger` from `~/lib/logger`; never raw `console.log`. Never log request bodies (contain scraped markdown).
- **Crypto**: use `crypto.subtle` or Web Crypto API. Never `Math.random()` for security.
- **No empty catch blocks** ‚Äî always show user-friendly error state.
