# AGENTS.md ‚Äî Parsify.dev

Single-purpose URL‚ÜíAgent product. Paste a URL ‚Üí **Jina Reader** (`r.jina.ai`) fetches and converts it to LLM-optimized markdown ‚Üí **DeepSeek** `deepseek-v4-flash` streams a summary. Both API keys live on the server. Built with TanStack Start v1 in full SSR mode, Hono at `/api/*`, deployed via Dokploy + Docker.

## Architecture

**TanStack Start v1 in full SSR mode** ‚Äî server renders pages; no static-only output. TanStack Router handles routing. Vite 7 is the build tool.

**Vite config quirk**: `rollupOptions.onwarn` suppresses `MODULE_LEVEL_DIRECTIVE` (Radix/TanStack "use client") and `UNUSED_EXTERNAL_IMPORT` (TanStack Start internals). Don't remove ‚Äî these are expected.

**API layer**: Hono app (`src/server/hono.ts`) configured with `{ strict: false }` (trailing slashes allowed), `.basePath('/api')`, mounted via a TanStack Start catch-all route (`src/routes/api/$.ts`). All HTTP methods forwarded to `app.fetch()`.

**Server routes:**
- `GET /api/health` ‚Äî returns `{ ok: true }`
- `POST /api/parse` ‚Äî Jina Reader proxy. Validates URL via Zod (SSRF guard rejects private/loopback/link-local hosts). Forwards user prompt as `X-Instruction`. Accepts optional `objective` (‚â§500 chars). Enforces 5 MB markdown response limit server-side.
- `POST /api/agent` ‚Äî DeepSeek SSE proxy ‚Üí plain text stream. Validates `markdown` (‚â§1 MB) and optional `prompt` (default: Chinese `'ËØ∑Áî®‰∏ÄÊÆµËØùÊÄªÁªìËøô‰∏™ÁΩëÈ°µÁöÑÊÝ∏ÂøÉÂÜÖÂÆπ'`). Returns 500 if `DEEPSEEK_API_KEY` is missing. Rate-limited: 20 req / 15 min per IP.

**Validation**: Zod 4 schemas in `src/schemas/`. The parse schema has SSRF guard ‚Äî add test coverage in `src/__tests__/schemas/parse.test.ts` whenever it changes.

**Logging**: pino + hono-pino. Logger configured in `src/lib/logger.ts`. Level controlled by `LOG_LEVEL` env var (default `info`). pino redacts `*.apiKey`, `*.headers.authorization`, `*.headers.cookie`. Always use `c.var.logger` in Hono route handlers ‚Äî never `console.log`. (The global `onError` handler has a `console.error` fallback when logger is unavailable ‚Äî that's the only exception.)

**Rate limiting**: in-memory `hono-rate-limiter` on `/api/agent` (20 req / 15 min per IP). Single-container deploy assumption; replace store if multi-instance.

## Code style (Biome)

- 2-space indent, 100 char width, single quotes, JSX double quotes, semicolons always, trailing commas (ES5), LF
- `assist.organizeImports: "on"` ‚Äî imports are auto-organized on save
- Enabled rules: `noUnusedVariables`/`noUnusedImports` (error), `noLabelWithoutControl` (off), `noForEach`/`useLiteralKeys` (off), `noDangerouslySetInnerHtml` (off), `noNonNullAssertion`/`noArrayIndexKey`/`noAssignInExpressions`/`noExplicitAny`/`noImplicitAnyLet` (off)
- TypeScript: strict, `noUncheckedIndexedAccess`, `noPropertyAccessFromIndexSignature`
- `import type` for type-only imports, explicit return types on exported functions

## TypeScript gotcha

`noPropertyAccessFromIndexSignature` means `process.env.NODE_ENV` fails ts(4111). Use bracket notation: `process.env['NODE_ENV']`. Same for any index-signature property access.

## Deploy (Dokploy + Docker)

Push to `main`. Docker build (`Dockerfile`):
1. `bun install --frozen-lockfile`
2. `bun run build` ‚Üí `dist/server/server.js` + `dist/client/`
3. `CMD ["bun", "run", "start"]` ‚Äî `srvx` on port 5173

**Required env vars:**
| Var | Required | Purpose |
|---|---|---|
| `PUBLIC_ORIGIN` | yes | Canonical origin ‚Äî used for CORS `origin` |
| `DEEPSEEK_API_KEY` | yes | `/api/agent` returns 500 without it |
| `JINA_API_KEY` | no | Without it, `/api/parse` uses anonymous tier (20 RPM) |
| `LOG_LEVEL` | no | pino level (default `info`): `trace`, `debug`, `info`, `warn`, `error` |

Note: `.env.example` uses `PUBLIC_ORIGIN=http://localhost:5173` matching the dev server default port.

## Security

**Server-side key invariant**: `DEEPSEEK_API_KEY` and `JINA_API_KEY` are read from `process.env` only. Never sent from browser, never logged, never persisted (no DB, no cache, no file), and never echoed in responses. pino redaction (`*.apiKey`, `*.headers.authorization`, `*.headers.cookie`) is defense-in-depth.

- **SSRF guard**: `parseRequestSchema` rejects 127.x, 10.x, 172.16‚Äì31.x, 192.168.x, 169.254.x, ::1, localhost.
- **Rate limiting**: `/api/agent` capped at 20 req / 15 min per IP.
- **Logging**: use `c.var.logger` in Hono handlers; never `console.log`. Never log request bodies (contain scraped markdown).
- **Crypto**: use `crypto.subtle` or Web Crypto API. Never `Math.random()` for security.
- **No `dangerouslySetInnerHTML`** without sanitization (static analytics scripts in root layout are the only exception).
- **No empty catch blocks** ‚Äî always show user-friendly error state.
